Herramientas disponibles

Privantix ofrece tres utilidades portables: inspección estructural de datasets, auditoría de permisos y generación de catálogos de inventario.

Privantix Inspector
Análisis técnico de repositorios

Escanea datasets tabulares, detecta estructura y genera metadata útil para auditoría, documentación e integración de fuentes.

Metadata técnica

Encoding, delimitador, encabezado, recuento de filas y columnas.

Perfilado de columnas

Tipos inferidos, nulos, muestras y reglas básicas de validación.

Privantix ACL Audit
Auditoría de permisos

Revisa ACL y permisos de carpetas para identificar exposición innecesaria, validar grupos confiables y documentar evidencia técnica.

ACL y permisos

Análisis de carpetas compartidas y extracción de permisos efectivos.

Trusted groups

Clasificación de grupos confiables y exportación de hallazgos en JSON y CSV.

Privantix Catalog
Inventario de datasets

Transforma los resultados del Inspector en un inventario estructurado de datasets, con visibilidad de columnas, metadata e indicadores de gobernanza.

Inventario de datasets

Catálogo de datasets y columnas con metadata técnica.

Indicadores de gobernanza

Datos sensibles, checksum, ACL y metadata de seguridad.

Formatos soportados
CSV XLSX ZIP / 7z / RAR Parquet / DAT / RDAT Repositorios locales y compartidos

El roadmap contempla conectividad futura a catálogos, bases de datos y monitoreo continuo de repositorios.

Ejemplos CLI

Comandos por herramienta:

# Inspector — escanear repositorio
privantix-inspector scan --path ./repository --output ./results

# Catalog — inventario desde Inspector
privantix-catalog --input ./results/analysis.json --output ./catalog

# ACL Audit — auditoría básica
privantix-acl-audit --path ./repository --output ./results

# ACL Audit — con grupos de confianza
privantix-acl-audit --path ./datos --output ./audit --trusted-groups ./trusted-groups.json

# ACL Audit — con usuarios dados de baja
privantix-acl-audit --path ./datos --output ./audit --revoked-users ./revoked-users.json

Ejemplos completos
# Inspector: escaneo básico
privantix-inspector scan --path ./datos --output ./reporte

# Catalog: generar catálogo desde Inspector
privantix-catalog --input ./reporte/analysis.json --output ./catalog

# Inspector: modo gobierno (sin muestras de datos)
privantix-inspector scan --path ./datos --output ./audit --hide-samples

# Inspector: con seguridad y checksum
privantix-inspector scan --path ./repositorio --output ./audit --security --checksum

# Inspector: extensiones y filas personalizadas
privantix-inspector scan --path ./datos --config config.yaml --extensions csv,xlsx,parquet --max-sample-rows 500

# Inspector: solo carpeta raíz (no recursivo)
privantix-inspector scan --path ./datos --recursive=false

# ACL Audit: auditoría básica
privantix-acl-audit --path ./carpetas --output ./output

# ACL Audit: con grupos de confianza y nombre de salida
privantix-acl-audit --path ./datos --output ./audit --trusted-groups ./trusted-groups.json --output-name ProyectoX_ACL

# ACL Audit: con usuarios dados de baja
privantix-acl-audit --path ./datos --output ./audit --revoked-users ./revoked-users.json

# ACL Audit: grupos confiables + usuarios dados de baja
privantix-acl-audit --path ./datos --output ./audit --trusted-groups ./trusted-groups.json --revoked-users ./revoked-users.json
Grupos de confianza y usuarios dados de baja (ACL Audit)
Clasificación de principios y estado de cumplimiento

Con --trusted-groups defines usuarios y grupos autorizados. Con --revoked-users defines usuarios dados de baja que no deberían tener acceso. El auditor clasifica: compliant (solo confiables), non_compliant (principales externos), critical (usuarios dados de baja con acceso).

Ejemplo trusted-groups.json
{
  "groups": [
    {
      "name": "equipo_datos",
      "members": [
        "CORP\\jperez",
        "CORP\\mgarcia",
        "CORP\\lrodriguez"
      ]
    }
  ]
}
Ejemplo revoked-users.json
{
  "users": [
    "CORP\\jperez",
    "CORP\\mgarcia",
    "lrodriguez"
  ]
}
Salida
  • trusted_principals: principales en los grupos confiables
  • outside_principals: principales fuera de los grupos
  • revoked_principals: usuarios dados de baja con acceso
  • compliance_status: compliant / non_compliant / critical

El matching acepta principal completo (CORP\jperez) o solo usuario (jperez). Case-insensitive.

Privantix y las prácticas DAMA-DMBOK

Las herramientas Privantix se alinean con dominios del marco DAMA-DMBOK para gobierno y gestión de datos.

Privantix Inspector
Dominios DAMA
  • Metadata Management: extracción de metadata técnica (encoding, delimitador, tipos)
  • Data Discovery: escaneo y descubrimiento de datasets en repositorios
  • Data Quality: reglas de validación, detección de nulos y anomalías
  • Data Profiling: perfilado de columnas, tipos inferidos, estadísticas
Privantix ACL Audit
Dominios DAMA
  • Data Security Management: auditoría de permisos y ACL
  • Data Governance: validación de grupos confiables y cumplimiento
  • Access Control: identificación de exposición y principales con acceso
Privantix Catalog
Dominios DAMA
  • Metadata Management: inventario estructurado de metadata
  • Data Governance: indicadores de gobernanza y datos sensibles
  • Data Inventory: catálogo de datasets y columnas
  • Data Architecture (parcial): visibilidad de estructura y ubicación
DAMA-DMBOK
El Data Management Body of Knowledge define dominios de gobierno de datos. Privantix aporta capacidades prácticas para Metadata, Discovery, Quality, Profiling, Security y Governance.

Metodologías y estándares aplicados

Privantix se diseña “privacy & security by design” y traduce marcos reconocidos en controles, workflows y reportes accionables.

Privacidad y cumplimiento
Cómo llevamos marcos de privacidad a acciones concretas
  • Privacy by Design: minimización, propósito, control y trazabilidad desde el inicio.
  • DPIA/PIA: evaluación de impacto; reportes y evidencias reutilizables.
  • NIST Privacy Framework: inventario, gobernanza, control y comunicación de riesgo.
  • ISO/IEC 27701: extensión de privacidad sobre un SGSI (alineable a ISO 27001).
  • Principios de minimización: “need-to-know”, “least privilege” y retención.
Seguridad y gestión de controles
Controles, hardening y gestión de riesgo
  • ISO/IEC 27001: controles de seguridad, auditoría y mejora continua.
  • NIST Cybersecurity Framework: Identify–Protect–Detect–Respond–Recover.
  • CIS Controls: priorización de controles prácticos para reducir superficie de exposición.
  • OWASP: buenas prácticas para apps y APIs (incl. Top 10).
  • Threat modeling: STRIDE (seguridad) y LINDDUN (privacidad).
Gobierno de datos
Estructura, roles y calidad
  • DAMA-DMBOK: dominios de gobernanza, calidad y metadata.
  • COBIT: alineación de TI con objetivos de negocio y controles.
  • ITIL: gestión de servicios y operación (incidentes, cambios, problemas).
  • Data catalog / metadata: inventario y linaje para trazabilidad.
  • Data quality: reglas, métricas y evidencias de calidad.
Anonimización y riesgo de reidentificación
Técnicas estadísticas y prácticas seguras de compartición
  • Five Safes: safe people, projects, settings, data, outputs.
  • k-anonymity, l-diversity, t-closeness: evaluación de riesgo por atributos cuasi-identificadores.
  • Pseudonimización y enmascaramiento: reglas reproducibles y auditables.
  • Data sharing agreements: principios de propósito, minimización y control.
  • Auditoría: evidencias de transformación y versionado de datasets.
De marco a herramienta
Privantix convierte estos marcos en escaneos repetibles, políticas configurables, reportes de evidencia y un roadmap claro de capacidades.

Inteligencia Artificial y protección de datos

Privantix incorpora IA de forma responsable para detectar riesgos, proteger información y habilitar el uso seguro de datos.

IA responsable (Responsible AI)
  • Principios de transparencia y explicabilidad.
  • Evaluación de sesgos y riesgos algorítmicos.
  • Uso ético de datos para entrenamiento.
  • Supervisión humana en decisiones críticas.
Detección inteligente de información sensible
  • NLP para identificar datos sensibles en texto libre.
  • Clasificación semántica de documentos.
  • Detección contextual más allá de patrones.
  • Modelos adaptables a español y LATAM.
Evaluación de riesgo para datasets usados en IA
  • Detección de atributos sensibles antes del entrenamiento.
  • Prevención de fuga de datos en modelos.
  • Evaluación de riesgo de reidentificación.
  • Buenas prácticas para datasets seguros.
Seguridad en modelos y LLM
  • Prevención de exposición de datos en prompts.
  • Control de información sensible en respuestas.
  • Protección frente a prompt injection.
  • Auditoría de uso y trazabilidad.
IA como aliada de la privacidad:
Privantix utiliza inteligencia artificial para aumentar la capacidad de detección, reducir errores humanos y permitir el uso seguro de los datos sin comprometer la privacidad.
